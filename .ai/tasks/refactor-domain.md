# Task: Refactor Domain

## Purpose

Safely refactor a library or domain while respecting boundaries.

## Steps

1. Identify the scope and impact of the refactor.
2. Use Nx commands to move or update code.
3. Update tags and dependencies as needed.
4. Run all tests and boundary checks.

## Acceptance Criteria

- No boundary violations or regressions.
- All affected code is tested and documented.
